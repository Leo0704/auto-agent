老板好！这是开发规范指南。

---

## 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 | 小驼峰 | `userName`, `orderList` |
| 常量 | 大驼峰或全大写 | `MaxSize`, `MAX_SIZE` |
| 函数/方法 | 小驼峰/大驼峰 | `getUserList`, `GetUserList` |
| 类/结构体 | 大驼峰 | `UserService`, `OrderController` |
| 文件名 | 小写+下划线/短横线 | `user_service.go`, `user-service.ts` |

---

## 注释规范

- 注释应该解释"为什么"，而不是"是什么"
- 复杂逻辑才需要注释
- 公共 API 需要文档注释
- 不要保留注释掉的代码

---

## 函数规范

- 单一职责：一个函数只做一件事
- 参数不超过 3-4 个，多了用结构体
- 早点返回，减少嵌套

---

## 安全规范

- 永远不要信任用户输入
- 使用参数化查询，不要字符串拼接 SQL
- 不要在日志中打印密码、密钥、token
- 不要在响应中返回敏感字段

---

## 性能规范

- 避免 N+1 查询
- 大表查询必须有分页
- 不要在循环中执行数据库查询
- Goroutine/线程要有退出机制

---

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 参数错误 | 返回友好提示 |
| 业务错误 | 返回友好提示 |
| 系统错误 | 记录日志，返回通用错误 |
| 外部服务错误 | 记录日志，可重试 |

---

## 日志规范

| 级别 | 使用场景 |
|------|---------|
| Error | 系统错误、外部服务调用失败 |
| Warn | 异常但可恢复的情况 |
| Info | 关键业务节点 |
| Debug | 开发调试信息 |
